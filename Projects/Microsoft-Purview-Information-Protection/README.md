# Microsoft Purview Information Protection & Data Classification

## Overview

This project demonstrates the implementation of Microsoft Purview Information Protection capabilities within a Microsoft 365 environment. The lab focuses on identifying, classifying, monitoring, and protecting sensitive information using Microsoft Purview's compliance and data governance features.

## Objectives

- Configure Microsoft Purview Information Protection
- Implement data classification and protection controls
- Discover and monitor sensitive information
- Apply sensitivity labels and compliance policies
- Strengthen data governance and regulatory compliance

## Technologies Used

- Microsoft Purview
- Microsoft 365
- Microsoft Entra ID
- Data Loss Prevention (DLP)
- Sensitivity Labels
- Content Explorer
- Data Explorer
- Optical Character Recognition (OCR)

## Key Activities

- Created Custom Sensitive Information Types (SITs)
- Implemented Fingerprint-based Classification
- Configured Exact Data Match (EDM) Classifiers
- Created Trainable Classifiers
- Configured Role-Based Access Control (RBAC)
- Enabled Content Explorer and Data Explorer
- Configured OCR-based Classification
- Created and Published Sensitivity Labels
- Performed Information Protection Monitoring and Validation

## Skills Demonstrated

- Data Classification
- Information Protection
- Compliance Management
- Data Governance
- Data Loss Prevention (DLP)
- Sensitivity Labeling
- Security Monitoring
- Microsoft Security & Compliance Administration

## Outcome

This project provided hands-on experience implementing enterprise information protection controls, improving visibility of sensitive data, and applying security and compliance best practices within Microsoft 365 environments.
